# Hybrid ICU Occupancy Forecasting

This repository contains the code for a short-term ICU bed occupancy forecasting project.  
We develop a **hybrid model** that combines:
- an **admission model** (predicting new ICU inflow), and  
- a **length-of-stay (LoS) model** (predicting remaining stay for current patients),
- A **hybrid model** combining both inflow and outflow signals to generate ICU occupancy and occupancy-change predictions  

to generate 3-, 5-, and 7-day forecasts of ICU occupancy and occupancy change.

All code is implemented in Jupyter notebooks.


## 1. Background

Reliable short-term ICU occupancy forecasting is crucial for hospital capacity planning, especially under fluctuating admission patterns and limited critical-care resources.  

This project was conducted as part of the **ST5188 Capstone Project** at the National University of Singapore (NUS), using the **MIMIC-IV** database to construct and evaluate a hybrid ICU occupancy prediction framework.

## 3. Data Requirements

### Source Dataset
This source dataset is:
- **MIMIC-IV v3.1**  
  Alistair Johnson et al https://physionet.org/content/mimiciv/3.1/ Oct. 11, 2024. Version: 3.1
Specifically, we used the following data tables:
- From the *hosp* module:
  - `admissions`: include hospital-wide admission and discharge records for all patients. Key column is *hadm_id*(admission id).
  - `patients`: demographic and background information of patients. Key column is *subject_id*.
  - `d_icd_procedures`,`d_icd_diagnosis`: demension tables for procedures and diagnosis
- From the *icu* module:
  -  `icu_stays`: admission and discharge information for all ICU patients, with anchor years randomly assigned.  Key columns is *stay_id*.
  -  `d_items`: dimension table describing itemid. Key columns is *item_id*.
  -  `chartevents`: charted items recorded during the ICU stays. Linked to `d_items` by *item_id*, `patients` by `subject_id`, `admission` by *hadm_id*.
  - `inputevents`: documented events of continuous infusions or intermittent administrations
  - `procedureevents`: procedure documented during ICU stays.

These data tables are expected to be downloaded into Dataset/ folder first before merging and preprocessing.
  - `admissions`, `patients`, `icu_stays` are used to provide patient demographics as well as the information of their admission and discharge (i.e. to generate `icu_patients.csv`)
  - `d_items`, `chartevents` are used to create `selected_chartevents.csv` which is essential for the subsequent LoS predictions.
  - The remaining tables are also used as features for LoS prediction.
  - `train_los` is generated by Preprocessing.ipynb
  - `procedures_embeddings.parquet` is created as embeddings on the list of procedures and joined with `train_los` in Training.ipynb

## 4. Workflow and Notebook Execution Order

Run the notebooks in the following order to fully reproduce the pipeline:

1. DataMerging.ipynb
   - Select and merge relevant tables from the source datasets.
   - Perform basic feature selection from chartevents and aggregate events at the daily level.
   - Address the anchor_year issue and apply random year assignment based on the provided year groups.
   - Deliverables: *icu_patients.csv*, *selected_chartevents*, and inflow/outflow calculation outputs.
2. Preprocessing.ipynb
   - Load the selected_chartevents, icu_patients generated by DataMerging.ipynb
   - Pivot vital signs readings into columns
   - Obtain actual age from anchor age
   - Deal with physiological outliers of vital signs using MICE algorithm
   - Deal with missing values using MICE algorithm
   - Add in features related to vasopressor drugs intake, RTT (Renal Replacement Treatment)
   - Compute remaining LoS as target
4. Create_Embedding.ipynb
   - Create embeddings from list of procedures from BioClinicalBERT model https://huggingface.co/emilyalsentzer/Bio_ClinicalBERT
6. Training.ipynb
   - Load training data and join with embeddings created from Create_Embedding.ipynb
   - Correlation analysis
   - Create PCA on embeddings for dimensionality reduction
   - Perform time based split
   - Construct sklearn Pipeline for feature preprocessing (minmaxscaler, robustscaler, one hot encoding etc)
   - Initial training for model selection
   - Hyperparameter tuning
   - SHAP importance
   - Analyzing text data for additional insights
8. utils/date_adjustments.ipynb: mid-term adjustment of assigned dates to ensure consistent predictions across the two sub-models
9. Admission_forecast_attempts.ipynb
   - Re-generate ICU admissions and discharges from *icu_patients.csv* without distinguishing between ICU departments.
   - Perform time series forecasting using moving average, ARIMA, and exponential smoothing methods supported by `darts`.
   - Evaluate, visualize, and conduct residual diagnostics to assess model performance.
10. Test_Set_Imputation.ipynb
   - Impute missing patient-day rows in the test set
   - Ensure fully aligned timelines between LoS and admission models
   - Prepare complete hybrid-model input data
11. Hybrid Model for ICU Occupancy Prediction.ipynb
   - Combine predicted inflow and remaining LoS
   - Generate occupancy and occupancy-change forecasts
   - Evaluate hybrid model performance
